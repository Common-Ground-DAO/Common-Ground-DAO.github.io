      <p class="lead mb-8">Everyone's heard of token gating, but what is it, and how do you leverage it on Common Ground?</p>

      <img src="/assets/images/blog/web3-social-roundup-5.webp" alt="Token gate anything on Common Ground with Roles" class="mb-8">

      <h2 class="heading-4 mb-4">What is Token Gating?</h2>
      <p class="mb-6">In crypto/web3, "token gating" is synonymous with exclusivity. Token gating is a popular mechanism that unlocks new ways to engage and reward communities.</p>

      <p class="mb-6">Token gating is an access control mechanism that hinges on the ownership of specific cryptographic tokens or non-fungible tokens (NFTs). Access to certain services, content, or experiences is exclusively reserved for those who possess these tokens. This concept is an important part of Web3, offering a unique blend of exclusivity and community engagement.</p>

      <p class="mb-6">Creating the feeling of "wanting to have it" when launching an NFT collection is the entire battle. It can be because of the unique art style, because of exclusivity, because it unlocks special privileges. Exclusivity is where Common Ground becomes extremely interesting.</p>

      <p class="mb-6">At a technical level, token gating operates through smart contracts on blockchain platforms. These contracts autonomously verify the ownership of the required tokens, granting or restricting access accordingly. This verification is decentralized and transparent. With token gating, communities can cultivate exclusive experiences, whether it be access to premium content, special events, or unique services, thereby adding a layer of value and exclusivity to token ownership.</p>

      <p class="mb-8">And remember: first you build the community and market towards a generalised audience, and second, after the mint (launch of the collection), you market towards your members. This is important and increases the value of being and becoming a member.</p>

      <h2 class="heading-4 mb-4">How Token Gating works on Common Ground</h2>
      <p class="mb-6">On Common Ground, access control and token gating works by using Roles. Whenever you post an article, or create a chat, or start a call, you can set which roles can see it, read it, or participate.</p>

      <p class="mb-6">It's quite simple, you create a role, decide who can have it and what it can do.</p>

      <p class="mb-4">There are currently 3 options:</p>
      <ul class="mb-6">
        <li>Roles can only be assigned to someone by an Admin</li>
        <li>Roles can be claimed by anyone</li>
        <li>Roles can be claimed by specific token holders</li>
      </ul>

      <p class="mb-6">That means that in just a few clicks, you can create exclusive spaces, content and events by using roles.</p>

      <p class="mb-6">You can see 3 fixed roles: Admin, Guests, and Members.</p>

      <p class="mb-6">Admins are all-powerful and can set up everything in a community, distribute carefully.</p>

      <p class="mb-6">Guests are any users who are visiting your community but haven't joined yet.</p>

      <p class="mb-6">Members are all users who have joined your community.</p>

      <p class="mb-6">On top of that, you can create Custom Roles, with custom requirements and permissions.</p>

      <p class="mb-6">Be aware that if you give full permissions to Guests, then it doesn't matter if you limit permissions of Members or custom Roles, because the weakest role (Guests) can already access it.</p>

      <p class="mb-8">You can use any of 9 EVM Blockchains for gating already, and we'll be adding more soon!</p>

      <h2 class="heading-4 mb-4">Exclusive Posts</h2>
      <p class="mb-8">When you create a post, you can set which Roles can read it, and which can only preview it (they'll see the tile, but when they click it, they'll see the gate). For example, a post that only users with a specific role can Read, but all members can Preview.</p>

      <h2 class="heading-4 mb-4">Exclusive Chats</h2>
      <p class="mb-8">When you create a chat, you can set which roles can chat, which can only read, and which can only see a preview (They'll see the chat name on the sidebar with a lock symbol). You can create chats that nobody can access or see except specific team members.</p>

      <h2 class="heading-4 mb-4">Exclusive Calls</h2>
      <p class="mb-8">When creating a call, you can limit which roles can join. This is an advanced permission so Admins can choose who can do it.</p>

      <h2 class="heading-4 mb-4">Encountering a Gate</h2>
      <p class="mb-6">When users click on a post or chat which is Gated, they will encounter a gate screen. This shows the user that the content or chat is locked, and that a role is required. The gate shows which role is Token Gated, and what tokens are required.</p>

      <p class="mb-6">Once you hold the required tokens in your wallet which you have connected to Common Ground, the role will become claimable to you automatically. CG is smart that way!</p>

      <p class="mb-8">If you lose access to those tokens (by selling them, for example), you will lose access to the Role, and therefore the content. That creates a trustless environment where you can be sure people have the right tokens.</p>

      <h2 class="heading-4 mb-4">Summary</h2>
      <p class="mb-6">Common Ground has made building communities with Token Gating simpler with Roles. You can reward your community with exclusive content, chats, and live events by using Token Gated Roles.</p>

      <p class="mb-6">It allows for a streamlined, tidy, and engaged community. We'll be adding more Blockchains, more Gating mechanics, and more content options as we continue building Common Ground, allowing for increasingly rich types of communities and roles to emerge.</p>
